# 基础语法


## 关于默认的6个成员函数以及移动语义

移动语义的本质是获取所有权。


默认构造、默认拷贝构造、默认拷贝赋值、默认移动构造、默认移动赋值、默认虚构。

由于，默认的拷贝构造、默认拷贝赋值、默认移动构造、默认移动赋值只是简单的绑定指针，而没有把原始对象置nullptr ，导致两个对象指向同一个内存，会造成 double free

```cpp
#include <iostream>

class Buffer{
    char* data_;
    size_t size_; 
public:
    Buffer(size_t n): data_(new char[n]), size_(n){
        std::cout << "Buffer constructor, size: " << size_ << std::endl;
    }
    // 拷贝构造
    Buffer(const Buffer& other): data_(new char[other.size_]), size_(other.size_){
        std::cout << "Buffer copy constructor, size: " << size_ << std::endl;
        std::copy(other.data_, other.data_ + size_, data_);
    }
    // 拷贝赋值运算符
    Buffer& operator=(const Buffer& other){
        std::cout << "Buffer copy assignment operator, size: " << other.size_ << std::endl;
        if(this != &other){
            delete[] data_; // 释放当前资源
            size_ = other.size_;
            data_ = new char[size_]; // 分配新资源
            std::copy(other.data_, other.data_ + size_, data_); // 复制数据
        }
        return *this;
    }
    // 必须实现移动构造函数和移动赋值运算符，否则会调用默认的拷贝构造函数和拷贝赋值运算符，导致 double free 错误
    Buffer(Buffer&& other) noexcept : data_(other.data_), size_(other.size_) {
        std::cout << "Buffer move constructor, size: " << size_ << std::endl;
        other.data_ = nullptr; // 转移资源所有权
        other.size_ = 0;
    }

    // 移动赋值运算符
    Buffer& operator=(Buffer&& other) noexcept {
        std::cout << "Buffer move assignment operator, size: " << other.size_ << std::endl;
        if (this != &other) {
            delete[] data_; // 释放当前资源
            data_ = other.data_; // 转移资源所有权
            size_ = other.size_;
            other.data_ = nullptr; // 转移资源所有权
            other.size_ = 0;
        }
        return *this;
    }
    // 必须手写析构函数
    ~Buffer(){
        std::cout << "Buffer destructor, size: " << size_ << std::endl;
        delete[] data_;
    }
};

int main(){
    Buffer buf1(10); // 调用构造函数
    Buffer buf2 = buf1; // 调用拷贝构造函数
    Buffer buf3(20);
    buf3 = buf1; // 调用拷贝赋值运算符

    Buffer buf4 = std::move(buf1); // 调用移动构造函数
    Buffer buf5(30);
    buf5 = std::move(buf2); // 调用移动赋值运算符

    return 0;
}
```

```text
所以移动语义的本质是获取原有对象的所有权？                                                                               
                                             
● 是的，移动语义的本质就是转移所有权。                                                                                     
                                                                                                                         
  更精确地说：                                                                                                             
                                                                                                                           
  ▎ 移动 = 从"即将消亡"的对象那里窃取资源，而不是复制资源。                                                                
                                                                                                                           
  ---
  所有权转移的三个动作

  以你的代码为例：

  Buffer(Buffer&& other) noexcept
      : data_(other.data_)   // 1. 我拿走你的指针
      , size_(other.size_)
  {
      other.data_ = nullptr; // 2. 你的指针清空（你不再拥有它）
      other.size_ = 0;       // 3. 你变成"空壳"
  }

  ┌─────────────────────────┬──────────────────────────┐
  │          动作           │           含义           │
  ├─────────────────────────┼──────────────────────────┤
  │ data_(other.data_)      │ 新对象获得资源所有权     │
  ├─────────────────────────┼──────────────────────────┤
  │ other.data_ = nullptr   │ 原对象放弃所有权         │
  ├─────────────────────────┼──────────────────────────┤
  │ 析构时 delete[] nullptr │ 空壳析构是安全的，无操作 │
  └─────────────────────────┴──────────────────────────┘

  ---
  与拷贝的对比

  拷贝：A ──→ [内存]        深拷贝后：A ──→ [内存A]
                                      B ──→ [内存B]（新分配）

  移动：A ──→ [内存]        移动后：  A ──→ nullptr（放弃）
                                      B ──→ [内存]（原来的）

  拷贝是"复印"，移动是"搬家"。

  ---
  为什么要有移动语义

  核心是避免不必要的深拷贝。对象在以下场景会"即将消亡"，资源完全可以直接转移：

  Buffer make_buffer() {
      Buffer tmp(1024);
      return tmp;           // tmp 即将销毁，直接把内存转给调用方
  }

  Buffer b = make_buffer(); // 移动构造，零次内存分配+复制

  如果没有移动语义，这里会 new 一块 1024 字节的新内存，把数据 memcpy 过去，再 delete 旧的——完全是浪费。

  ---
  一句话：移动语义让 C++ 能安全地说"这个对象我不要了，你拿去用"，省掉深拷贝的开销，同时通过置 nullptr 保证所有权唯一，避免
  double free。
```
