```text
事实上，从 本地采用 ffmpeg -re -i video.mp4 -c:v libx264 -preset ultrafast -c:a aac -f rtsp rtsp://127.0.0.1:554/live/test 采用 h264 解码推流到 ZLM ，浏览器端通过：
http://localhost:3000/ 

通过刷新由/index/api/getMediaInfo?secret=llA7O866EEum4lM7Rssbnnw4cZsfFAlo&vhost=__defaultVhost__&schema=hls&app=live&stream=test向 ZLM 服务端请求，其中ZLM 服务从启动再到响应的全流程，精确到函数级别

```

```text
控制面：浏览器 → getMediaInfo → ZLM → 返回 JSON（只有元数据，没有视频数据）
数据面：浏览器播放器 → 请求 HLS/HTTP-FLV/WebRTC → ZLM → 返回媒体数据 → 解码播放

```
macos 场景：
schema=hls，所以浏览器拿到 isRecorddingHLS:true 播放器回去请求:

```text
GET /live/test/hls.m3u8
GET /live/test/xxxxx.ts

```
第一段:ffmpeg 推流进 ZLM -> 产出 HLS 文件

> HLS模块切完 .ts 切片、更新好hls.m3u8后，文件落盘或存放内存，等浏览器来取

```text
ffmpeg 发 RTP
  └─ RtspSession::onRecv(buf)
       └─ RtspSplitter::input(...)
            └─ RtspSplitter::onRecvHeader(...)
                 └─ RtspSession::onWholeRtspPacket(parser)   ← RTSP 信令处理
                      └─ handleReq_RECORD/SETUP ...

  └─ (RTP包到来) RtspSession::onRtpPacket(data, len)
       └─ RtpReceiver::handleOneRtp(...)                     ← 排序/重组
            └─ RtpReceiver::onRtpSorted(rtp, track_idx)
                 └─ RtspSession → (frame解码) → Frame 送给 muxer

  └─ MultiMediaSourceMuxer::inputFrame(frame)                ← 转封装中枢
       ├─ HlsRecorder::inputFrame(frame)                     ← 产出 .ts / .m3u8
       ├─ RtmpMediaSource (供 http-flv)
       └─ RtspMediaSource (供 rtsp 拉流)
```

第二段：浏览器请求 m3u8 ->ZLM 返回

```text
浏览器 GET /live/test/hls.m3u8
  └─ HttpSession::onRecv(buf)
       └─ HttpRequestSplitter::input(...)
            └─ HttpSession 解析出 URL: /live/test/hls.m3u8

  └─ NoticeCenter emit kBroadcastHttpRequest
       └─ 未命中 s_map_api（不是 /index/api/ 路径）
            └─ 进入 HttpFileManager::process(...)            ← 文件型 HTTP 处理
                 └─ 查找文件路径 (www/live/test/hls.m3u8)
                      └─ HttpSession::sendResponse(200, ...)
                           └─ Socket::send(...)              ← 把 m3u8 内容发回浏览器
```

第三段：浏览器请求 .ts -> ZLM 返回分片

```text
浏览器 GET /live/test/20260301_xxxx.ts
  └─ HttpSession::onRecv(buf)
       └─ 同上路径进 HttpFileManager::process(...)
            └─ 找到对应 .ts 文件
                 └─ HttpSession 发送文件内容（可能分块）
                      └─ Socket::send(...)                   ← 实际字节流发给浏览器

```

第四段:浏览器侧
```text
浏览器播放器（HLS.js / Video.js / 原生 <video>）
  └─ 收到 .ts 文件
       └─ demux（TS → H264 NAL + AAC）
            └─ MSE / WebCodecs 解码
                 └─ 渲染到 <video> 标签播放

```

事实上，getMediaInfo 和播放是独立的。

```text
前端代码逻辑（你的 localhost:3000）通常这样：

1. fetch('/index/api/getMediaInfo?...')   ← 控制面查询
   → 拿到 isRecordingHLS:true

2. 构造播放地址：
   http://127.0.0.1/live/test/hls.m3u8

3. video.src = 'http://127.0.0.1/live/test/hls.m3u8'
   或者 hls.loadSource(url)              ← 数据面拉流


ffmpeg 推流
    ↓ RTSP
ZLM RtspSession 收包
    ↓ RTP → Frame
MultiMediaSourceMuxer 转封装
    ↓ HLS 切片
文件/内存中的 .m3u8 + .ts
    ↓ HTTP 静态文件服务
浏览器 HLS 播放器拉取
    ↓ TS demux + 解码
最终播放

```



> getMediaInfo 本身不传视频，它只是表示这条流“存在且可播"，前端拿到这个信息后，播放器再去拉媒体数据



## ZLToolKit

1. thread
> 为什么每个项目都要有自己的一套组件，尤其是线程池、事件模式、调度 这几大类耦合在一起，其实阅读性很低。当然如果说它们是共通的还好，但是很多时候采用C++新特性，会把这几部分抽象。


```cpp
size_t TaskExecutorGetterImp::addPoller(const string &name, size_t size, int priority, bool register_thread, bool enable_cpu_affinity) {
    auto cpus = thread::hardware_concurrency();
    size = size > 0 ? size : cpus;
    for (size_t i = 0; i < size; ++i) {
        auto full_name = name + " " + to_string(i);
        auto cpu_index = i % cpus;
        EventPoller::Ptr poller(new EventPoller(full_name));
        poller->runLoop(false, register_thread);
        poller->async([cpu_index, full_name, priority, enable_cpu_affinity]() {
            // 设置线程优先级  [AUTO-TRANSLATED:2966f860]
            //Set thread priority
            ThreadPool::setPriority((ThreadPool::Priority)priority);
            // 设置线程名  [AUTO-TRANSLATED:f5eb4704]
            //Set thread name
            setThreadName(full_name.data());
            // 设置cpu亲和性  [AUTO-TRANSLATED:ba213aed]
            //Set CPU affinity
            if (enable_cpu_affinity) {
                setThreadAffinity(cpu_index);
            }
        });
        _threads.emplace_back(std::move(poller));
    }
    return size;
}
```

## WebApi

HTTP 请求打到 /index/api/xxx 以后的链路：
1. HttpSession 收到请求(在该连接所属的 EventPoller 线程)
2. 路由分发到WebApi.cpp 注册的 API 处理器(api_regist)
3. 处理器内部：
    - 先做参数校验、鉴权(secret\schema)
    - 再决定是同步直接返回还是异步任务(如跨模块查询、hook、播放器操作)
    - 结果统一塞进 JSON
    - 通过 HTTPresponse 回写给客户端
  

总的来说，WebAPi 充当的是 控制面API的聚合层(轻量Controller层)
- 不负责底层网络包(那是HttpSession)
- 不负责媒体处理细节(那是 MediaSOurce/Rtsp*/Rtmp*/WebRtc*)
- 它做的事：
    - 注册接口名与处理函数映射
    - 提供统一参数访问/错误码包装
    - 把请求扔给正确模块并回收结果

```cpp
template <typename Type>
class ServiceController {
public:
    using Pointer = std::shared_ptr<Type>;
    std::unordered_map<std::string, Pointer> _map;
    mutable std::recursive_mutex _mtx;

    void clear() {
        decltype(_map) copy;
        {
            std::lock_guard<std::recursive_mutex> lck(_mtx);
            copy.swap(_map);
        }
    }

    size_t erase(const std::string &key) {
        Pointer erase_ptr;
        {
            std::lock_guard<std::recursive_mutex> lck(_mtx);
            auto itr = _map.find(key);
            if (itr != _map.end()) {
                erase_ptr = std::move(itr->second);
                _map.erase(itr);
                return 1;
            }
        }
        return 0;
    }

    size_t size() { 
        std::lock_guard<std::recursive_mutex> lck(_mtx);
        return _map.size();
    }

    Pointer find(const std::string &key) const {
        std::lock_guard<std::recursive_mutex> lck(_mtx);
        auto it = _map.find(key);
        if (it == _map.end()) {
            return nullptr;
        }
        return it->second;
    }

    void for_each(const std::function<void(const std::string&, const Pointer&)>& cb) {
        std::lock_guard<std::recursive_mutex> lck(_mtx);
        auto it = _map.begin();
        while (it != _map.end()) {
            cb(it->first, it->second);
            it++;
        }
    }

    template<class ..._Args>
    Pointer make(const std::string &key, _Args&& ...__args) {
        // assert(!find(key));

        auto server = std::make_shared<Type>(std::forward<_Args>(__args)...);
        std::lock_guard<std::recursive_mutex> lck(_mtx);
        auto it = _map.emplace(key, server);
        assert(it.second);
        return server;
    }

    template<class ..._Args>
    Pointer makeWithAction(const std::string &key, std::function<void(Pointer)> action, _Args&& ...__args) {
        // assert(!find(key));

        auto server = std::make_shared<Type>(std::forward<_Args>(__args)...);
        action(server);
        std::lock_guard<std::recursive_mutex> lck(_mtx);
        auto it = _map.emplace(key, server);
        assert(it.second);
        return server;
    }

    template<class ..._Args>
    Pointer emplace(const std::string &key, _Args&& ...__args) {
        // assert(!find(key));

        auto server = std::static_pointer_cast<Type>(std::forward<_Args>(__args)...);
        std::lock_guard<std::recursive_mutex> lck(_mtx);
        auto it = _map.emplace(key, server);
        assert(it.second);
        return server;
    }
};
```

路由到文件管理器
```text
HttpSession 收到完整 HTTP 请求后
  └─ NoticeCenter emit kBroadcastHttpRequest
       └─ 未命中 s_map_api（非 /index/api/）
            └─ 继续走文件访问路径
                 └─ HttpFileManager::onAccessPath(sender, parser, cb)
```
onAccessPath 内部

```text
HttpFileManager::onAccessPath(sender, parser, cb)
  ├─ fullUrl = "http://" + Host + fullUrl()
  ├─ media_info = MediaInfo(fullUrl)       ← 解析 schema/vhost/app/stream
  ├─ file_path = getFilePath(parser, media_info, sender)
  │    ├─ 拼接 rootPath + url（如 www/live/test/hls.m3u8）
  │    ├─ NOTICE_EMIT(kBroadcastHttpBeforeAccess)  ← 触发 before_access hook
  │    └─ 返回绝对路径
  └─ 非目录 → 进入 accessFile(...)

```

accessFile 内部
```text
accessFile(sender, parser, media_info, file_path, cb)
  ├─ is_hls = end_with(file_path, "/hls.m3u8")     ← true
  ├─ 修正 media_info.schema = HLS_SCHEMA
  ├─ 修正 media_info.stream 移除 "/hls.m3u8" 后缀
  └─ canAccessPath(sender, parser, media_info, false, callback)
       ├─ 查找 cookie（HttpCookieManager::Instance().getCookie(...)）
       │    ├─ 命中且未过期 → 直接走鉴权成功路径
       │    └─ 未命中 → 触发鉴权事件
       │         └─ emitHlsPlayed(parser, media_info, invoker, sender)
       │              └─ NOTICE_EMIT(kBroadcastMediaPlayed, ...)  ← on_play hook
       │                   └─ auth_invoker("") → 鉴权通过，设置 cookie
       └─ 鉴权通过后 → 执行 callback("", cookie)
```

回调那: m3u8 内容来源决策(内存 vs 磁盘)
```text
callback("", cookie) 执行 response_file 分支：

情况 A：src（HlsMediaSource）已注册 → 走内存
  └─ src = attach._hls_data->getMediaSource()   ← 拿到内存 HlsMediaSource
  └─ response_file(cookie, cb, file_path, parser,
                   src->getIndexFile())          ← 直接用内存 m3u8 字符串响应！
       └─ invoker.responseFile(..., file_content, is_path=false)
            └─ (*this)(200, header, HttpStringBody(file_content))
                 └─ Socket::send(m3u8内容字节)  → 发回浏览器

情况 B：src 尚未注册 → 触发异步按需查找
  └─ MediaSource::findAsync(media_info, session, callback)
       └─ 找到 HlsMediaSource 后：
            └─ hls->getIndexFile([...](const string &file) {
                   response_file(cookie, cb, file_path, parser, file);
               })                               ← 等 m3u8 就绪后再响应

情况 C：磁盘文件
  └─ response_file(cookie, cb, file_path, parser)
       └─ invoker.responseFile(header, header, file_path,
                               use_mmap=false, is_path=true)
            └─ HttpFileBody(file_path)          ← mmap 或 fread 读文件
                 └─ Socket::send(文件内容)      → 发回浏览器
```

浏览器请求 .ts 分片

```text
浏览器 GET /live/test/2026-03-02/03/10-00_5.ts
  └─ HttpSession → HttpFileManager::onAccessPath(...)
       └─ getFilePath(...) → 绝对路径 www/live/test/2026-03-02/03/10-00_5.ts
       └─ accessFile(...)
            ├─ is_hls = false（.ts 不匹配 /hls.m3u8 后缀）
            ├─ File::fileExist(file_path) 检查文件是否存在
            └─ canAccessPath(...)
                 └─ 命中 cookie（前面 m3u8 已建立 cookie）
                      ├─ 刷新 cookie 有效期（kHlsCookieSecond = 60s）
                      └─ response_file(cookie, cb, file_path, parser)
                           └─ invoker.responseFile(header, header,
                                                   file_path, use_mmap=false)
                                └─ HttpFileBody(file_path)  ← 磁盘读 ts 文件
                                     └─ Socket::send(ts内容) → 发回浏览器

```

响应发送的最终路径
```text
HttpResponseInvokerImp::operator()(200, headerOut, body)
  └─ _lambad(200, headerOut, body)         ← 回到 HttpSession 的 invoker
       └─ HttpSession 构造 HTTP response 头
            └─ Socket::send(header + body) ← 写入 socket 缓冲区
                 └─ EventPoller epoll_wait 触发可写事件
                      └─ 字节流最终发出到浏览器
```
