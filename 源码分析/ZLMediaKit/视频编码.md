# 视频编码

> 常见的视频编码标准 H.264/AVC

关键词：技术特点(帧内预测Intra prediction、帧间预测 Inter prediction、变换量化、环路滤波 Deblocking filter、熵编码、网络抽象层(NAL)设计，适应网络传输)

## H.264

帧内预测
- 亮度块支持9种 4x4 预测模式和4种16x16预测模式
- 利用相邻已解码像素推测当前块，大幅压缩空间冗余
- 新增 8x8 帧内预测(Hight Profile)

高度灵活的帧间预测
- 可变块大小: 从16x16 到 4x4，共7种划分方式，可更精确匹配运动物体边缘
- 1/4 像素精度运动补偿: 通过6抽头滤波器插值出亚像素位置，显著提升运动估计精度
- 多参考帧: 允许在多个先前帧种搜索最佳匹配，对周期性运动、遮挡恢复很有效
- B帧 可作为参考帧，提高编码效率


整数DCT变换

- 采用 4x4 整数DCT，完全避免浮点误差，解码器可严格匹配
- 对色度分量和特定情况使用 2x2、8x8 变换
- 引入 Hadamard 变换 对直流系数二次变换，进一步压缩

环路去块滤波
- 在编码环内对重建图像进行自适应滤波，减弱块效应
- 基于边界强度和图像内容判断是否滤波，既提升主观质量，又为后续帧提供更为准确的参考


先进的熵编码
- CAVLC(上下文自适应变长编码): 基础使用，根据已编码语法元素切换码表
- CABAC(上下文自适应二进制算术编程): Main/High Profile 使用，相比 CALVC 再节省10～15%码率，但计算量更大


网络友好性: NAL 单元
- 将编码后的视频数据打包成NAL(网络抽象层)单元，明确区分VCL(视频编码层)和非VCL(参数集、SEI)
- 支持 数据分割，将重要数据(宏块类型、运动矢量)与纹理数据分开传输，丢包时可部分恢复


档次（Profile）与级别（Level）  
H.264 通过 Profile 定义工具集，通过 Level 定义分辨率、帧率、码率上限，兼顾不同设备能力。

| Profile	| 典型应用	| 关键特性 | 
| :--:    | :--:    | :--:    |
| Baseline	| 视频会议、移动设备	| I / P 帧，CAVLC，不支持 B 帧，低延迟 | 
| Main	| 标清数字电视	| 支持 B 帧、CABAC、隔行扫描 | 
| High	| 蓝光、高清网络视频	| 8×8 帧内预测、8×8 变换、自定义量化矩阵 | 
| High 10 / High 4:2:2 / High 4:4:4	| 专业制作、母带	| 更高位深（10bit）、更高色度采样 | 

Level 如 3.0（720×576@30fps）、4.0（2K×1080@30fps）、5.1（4K×2160@30fps）等。
