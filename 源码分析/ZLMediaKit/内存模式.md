## Buffer 体系，统一的数据缓冲区抽象
ZLM 用一个纯虚基类 Buffer 统一所有数据载体，彻底避免裸 char* 到处传递

```cpp
// ZLToolKit/src/Network/Buffer.h

class Buffer : public noncopyable {
public:
    using Ptr = std::shared_ptr<Buffer>;
    virtual char *data() const = 0;
    virtual size_t size() const = 0;
private:
    ObjectStatistic<Buffer> _statistic; // 实例计数（见第四层）
};
```

在此基础上派生了三个主要子类

1. BufferRaw,堆上裸内存
```cpp
class BufferRaw : public Buffer {
    // 内部维护 char* _data，自管生命周期
    void setCapacity(size_t capacity) {
        if (_capacity < 2 * 1024) {
            return; // 2K 以下复用，避免频繁 new/delete
        }
        if (2 * capacity > _capacity) {
            return; // 新大小超过一半也复用
        }
        delete[] _data;
        _data = new char[capacity];
    }
};

```
关键设计：小块内存不缩容，大块内存超过一半才缩容，减少 new/delete 次数。

2. BufferLingString，零拷贝的字符串视图
```cpp
class BufferLikeString : public Buffer {
    std::string _str;
    size_t _erase_head = 0; // 逻辑头部偏移
    size_t _erase_tail = 0; // 逻辑尾部偏移

    char *data() const override {
        return (char *)_str.data() + _erase_head; // 不拷贝，只移指针
    }
    size_t size() const override {
        return _str.size() - _erase_tail - _erase_head;
    }
};
```

## 第三层 ResourcePool 对象池
频繁 new/delete 是高性能服务的杀手。ZLM 用 ResourcePool 实现对象池，核心思想：shared_ptr 析构时不 delete，而是归还给池子

```cpp
// ZLToolKit/src/Util/ResourcePool.h

template <typename C>
class ResourcePool_l {
    std::shared_ptr<C> obtain2() {
        auto weak_self = _weak_self;
        return std::shared_ptr<C>(getPtr(), [weak_self](C *ptr) {
            auto strongPool = weak_self.lock();
            if (strongPool) {
                strongPool->recycle(ptr); // 归还而非删除！
            } else {
                delete ptr;              // 池子没了才真正释放
            }
        });
    }

    C *getPtr() {
        // 用 atomic_flag 自旋锁（无锁竞争时极快）
        auto is_busy = _busy.test_and_set();
        if (!is_busy) {
            if (_objs.empty()) ptr = _alloc(); // 池空了才 new
            else { ptr = _objs.back(); _objs.pop_back(); } // 从池中取
            _busy.clear();
        } else {
            ptr = _alloc(); // 竞争时直接 new，不阻塞
        }
        return ptr;
    }
};
```

生命周期:
```text
第一次 obtain2()
  → 池空 → new BufferRaw()  ←────────────────┐
  → 返回 shared_ptr（自定义 deleter）          │
                                               │
使用完毕，shared_ptr 引用计数降为 0             │
  → deleter 触发 → recycle() → 放回 _objs ────┘

第二次 obtain2()
  → 池非空 → 从 _objs 取出（无 new！）
```

应用
```cpp
// RTMP 包池
toolkit::ResourcePool<toolkit::BufferRaw> _packet_pool; // RtmpProtocol.h, FlvMuxer.h

// 音视频帧池
toolkit::ResourcePool<FFmpegFrame> _frame_pool;         // Transcode.h

// MP4/MPEG 读写缓冲池
toolkit::ResourcePool<toolkit::BufferRaw> _buffer_pool; // MP4Demuxer.h, MPEG.h

```

## 第四层：ObjectStatistic —— 实例数量追踪

每个核心类内部嵌入一个 ObjectStatistic<T> 成员，构造时 +1，析构时 -1

```cpp
// ZLToolKit/src/Util/util.h

template <class C>
class ObjectStatistic {
public:
    ObjectStatistic()  { ++getCounter(); }
    ~ObjectStatistic() { --getCounter(); }
    static size_t count() { return getCounter().load(); }
private:
    static std::atomic<size_t>& getCounter(); // 每个类型一个全局原子计数器
};
```

Buffer、BufferRaw、BufferLikeString、Frame、FrameImp 都内嵌了它：

```cpp
class Buffer {
    ObjectStatistic<Buffer> _statistic; // 不占逻辑空间，仅统计用
};
class FrameImp {
    ObjectStatistic<FrameImp> _statistic;
};

```

完整架构图
```text
用户代码
    │
    ▼
┌───────────────────────────────┐
│  Buffer / Frame / RtpPacket   │  ← 统一数据载体（shared_ptr 管理生命周期）
│  (BufferRaw / BufferLikeStr)  │
└──────────────┬────────────────┘
               │ obtain2() 从池中取，析构时归还
               ▼
┌───────────────────────────────┐
│      ResourcePool<T>          │  ← 对象复用池（避免频繁 new/delete）
│  atomic_flag 自旋锁，无阻塞   │
└──────────────┬────────────────┘
               │
               ▼
┌───────────────────────────────┐
│   系统 malloc / free          │  ← ENABLE_MEM_DEBUG 时被 Hook
│   MemCookie 头部注入统计信息  │  ← 按线程追踪内存分配量
└───────────────────────────────┘

横切关注点：
ObjectStatistic<T>  ← 每个核心对象嵌入，追踪实例数量
shared_ptr          ← 所有核心对象的生命周期管理
```
