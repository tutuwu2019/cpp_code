# 一些思考

> 有一些问题，我自己有答案，同时参考ai 的回答，看看是不是重合度高或者哪里有出路

```text
c++中，对了，我想问问，lambda 函数从功能性出发，适用于哪种语义场景开发呢？

但是，从项目的维护角度来讲，并不适合把 lambda 函数封装成 功能性函数(维护角度，包括功能性、易读性、风格统一等)，反而更多的是延迟类，比如注册回调函数


#################################################################
我的思考：
实际上，lambda 应该精简、易读(这就包含了易维护)
基于这些特点，实际上，用我粗俗的话来讲，并不适合做功能函数的编写更多的是做粘合剂(捕获上下文(变量)，完成逻辑处理)，比如注册函数(延迟计算、回调函数)，当然还有一种，不常见，配合STL,完成自定义操作。

注意：不适合做项目级的多线程与并发操作，如果做测试级，那完全够用的。(好处自然是按值或者按引用捕获外部val，避免传递繁琐的参数)

### ai 参考
从维护角度，lambda 更适合作为延迟执行的闭包（回调、线程任务、事件处理等），而不是作为通用功能性函数。功能性函数应该用普通函数或函数对象实现，以获得清晰的名字、可测试性和复用性。
```
